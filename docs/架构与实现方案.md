# opcode-pg-memory 架构与实现方案

> **版本**: v3.15.0 ｜ **测试**: 100/100 (9 suites) ｜ **编译**: TypeScript 严格模式  
> **运行时**: Bun (构建) + Node.js 18+ (运行) ｜ **数据库**: PostgreSQL + pgvector  
> **对齐标准**: OpenCode Plugin SDK 1.14.48 + Claude Code MCP 规范

---

## 一、项目概述

`opcode-pg-memory` 是 OpenCode 桌面端的 PostgreSQL 长期记忆插件，提供：

1. **长期记忆存储**：工具调用（observations）、实体（entities）、反思（reflections）全生命周期管理
2. **多策略检索**：语义搜索（pgvector）、关键词搜索（ILIKE）、全文搜索（tsvector BM25）、知识图谱遍历（entity graph walk）
3. **自主认知闭环**：会话结束后自动触发反思 → 生成行为规则 → 自动应用到 Agent
4. **上下文注入**：两路召回（关键词 + 语义）混合排序，注入 LLM system prompt
5. **知识图谱**：零 LLM 纯正则实体提取 + 关系构建 + 项目骨架感知

## 四、核心数据流

### 4.1 工具调用 → 长期记忆

```
用户触发工具调用
    │
    ▼
[tool.execute.before] ──► 记录工具输入摘要 → observations 表
    │
    ▼
[工具执行]
    │
    ▼
[tool.execute.after]
    ├── output-compressor ──► 压缩工具输出（3阶段）
    ├── causal-chain-detector ──► 检测 failure→fix 因果链
    ├── entity-extractor ──► 纯正则提取实体 → entity-store
    ├── short-term-memory ──► 加入短期记忆（当前会话）
    ├── async-embedder ──► 异步向量化
    └── observations 表 UPDATE ──► 更新输出摘要 + 重要性评分
```

### 4.2 记忆检索 (recall_memory)

```
recallMemory(input)
    │
    ├── resolveSessionId ──► 解析 session（缓存 5min TTL）
    ├── topicFusion ──► 查询 + 当前话题 embedding 融合 (7:3)
    │
    ├── parallelRetrieve ──► 并行分发 4 种策略
    │   ├── semantic-search   (pgvector <=> 余弦相似度)
    │   ├── keyword-search    (ILIKE + entity relation JOIN)
    │   ├── bm25-search       (tsvector @@ to_tsquery)
    │   └── graph-traversal   (entity 关系图遍历)
    │
    ├── mergeAndDeduplicate ──► 去重合并
    ├── calculateMultiDimensionalScores ──► 多维评分
    │   └── Relevance = 0.5×SemSim + 0.3/(1+RecencyDays) + 0.2×Importance
    ├── applyFilters ──► 置信度/重要性/层级过滤
    ├── aggregateConsecutiveSimilar ──► 连续同类工具聚合
    ├── crossEncoderRerank ──► 交叉编码器重排（可选）
    └── enrichWithContext ──► 附加 session/topic 上下文
```

### 4.3 System Prompt 注入

```
[experimental.chat.system.transform]
    │
    ▼
buildInjectionBlock(input)
    │
    ├── Injection Cache 检查 ──► systemHash 相同 → 直接返回缓存
    │
    ├── retrieveMemoriesForInjection ──► 两路召回
    │   ├── Short-term memory（零延迟，当前会话内）
    │   ├── Path A: keywordRecall（DB 关键词查询，始终执行）
    │   └── Path B: semanticRecall（pgvector，冷启动跳过）
    │       └── 全局降级（项目结果 <3 条时触发）
    │           ├── 全局 keywordRecall
    │           ├── 全局 reflections 检索
    │           └── 全局 entities 检索
    │
    ├── fetchActiveRules ──► 读取已应用的行为规则
    ├── getProjectSkeleton ──► 项目文件骨架（token 预算 150）
    │
    └── formatInjectionBlock ──► 格式化为 <pg_memory> XML 块
        └── merge 到 output.system[0]
```

### 4.4 自主反思闭环

```
[session.completed]
    │
    ▼
handleSessionCompleted
    ├── observations 计数检查（≥30 条触发）
    ├── off-peak hours 检查（默认凌晨 1-5 点）
    │
    ├── hindsightReflect ──► 启发式反思引擎（无需 LLM）
    │   ├── 分析 tool_status: failed → error_pattern
    │   ├── 分析 tool_name 频率 → workflow 优化
    │   └── 分析编辑-构建-测试序列 → tool_preference
    │
    ├── auto-apply（confidence ≥ 0.8 + action_plan 存在）
    │   └── applyReflection ──► 5函数管道
    │       ├── checkDuplicate ──► 已应用？跳过
    │       ├── checkCooldown ──► 7天内同类型？跳过
    │       ├── formatRule ──► 格式化为 rules.md bullet
    │       ├── writeRuleAtomic ──► 原子写入 ~/.config/opencode/rules.md
    │       └── markApplied ──► 标记 applied_at 时间戳
    │
    └── auto-skill（confidence ≥ 0.85 + 非 rule 类型）
        └── skill-writer ──► 生成 agentskills.io 兼容技能文件
```

## 五、数据库 Schema（核心表）

### 5.1 session_map — 会话映射
```sql
id                UUID PRIMARY KEY
opencode_session_id TEXT UNIQUE    -- OpenCode 外部 session ID
omo_task_id       TEXT             -- OmO 任务 ID
project_id        TEXT             -- 项目标识
model_context_limit INT            -- 模型上下文窗口大小
created_at        TIMESTAMPTZ
last_active_at    TIMESTAMPTZ
metadata          JSONB            -- SessionMapMetadata
```

### 5.2 observations — 观察记录
```sql
id                UUID PRIMARY KEY
session_map_id    UUID → session_map
topic_segment_id  UUID → topic_segments (nullable)
tool_name         TEXT             -- 工具名
tool_input_summary  TEXT           -- 输入摘要
tool_output_summary TEXT           -- 输出摘要（压缩后）
tool_status       TEXT             -- 'pending' | 'completed' | 'failed'
tool_error        TEXT             -- 错误信息
importance        INT DEFAULT 3    -- 1-5
embedding         vector(1024)     -- pgvector（维度从 config 读取）
causal_chain_id   UUID             -- 因果链 ID
causal_role       TEXT             -- 'cause' | 'fix'
metadata          JSONB            -- ObservationMetadata
created_at        TIMESTAMPTZ
```

### 5.3 entities — 知识图谱实体
```sql
id                UUID PRIMARY KEY
session_id        UUID → session_map
name              TEXT             -- 实体名
type              TEXT             -- 'file' | 'function' | 'class' | 'module'
tier              TEXT             -- 'permanent' | 'project' | 'session'
weight            INT              -- 出现频次
confidence        FLOAT
embedding         vector(1024)
first_seen_at     TIMESTAMPTZ
last_seen_at      TIMESTAMPTZ
```

### 5.4 relations — 实体关系
```sql
id                UUID PRIMARY KEY
source_entity_id  UUID → entities
target_entity_id  UUID → entities
relation_type     TEXT             -- 'belongs_to' | 'depends_on' | 'references' | 'implements' | 'uses'
confidence        FLOAT
session_id        UUID → session_map
```

### 5.5 reflections — 反思记录
```sql
id                UUID PRIMARY KEY
session_id        UUID → session_map
summary           TEXT
pattern_type      TEXT             -- 'error_pattern' | 'workflow' | 'tool_preference'
confidence        FLOAT
action_plan       JSONB            -- 结构化可执行模式
applied_at        TIMESTAMPTZ      -- 规则应用时间
metadata          JSONB            -- ReflectionMetadata
```

### 5.6 辅助表
- **semantic_cache** — 语义缓存（query_hash → response）
- **token_usage_log** — Token 使用日志
- **topic_segments** — 话题分段

## 八、关键设计决策

### 8.1 纯正则实体提取（零 LLM 开销）
`entity-extractor.ts` 使用纯正则表达式从工具输出中提取 `file/function/class/module` 实体，fire-and-forget 非阻塞。不使用 LLM，零 token 消耗。

### 8.2 启发式反思引擎（无需 LLM API）
`hindsight-reflect.ts` 100% 基于规则：分析 `tool_status: failed` → error_pattern，分析工具调用频率 → workflow，分析编辑-构建-测试序列 → tool_preference。不依赖任何 LLM API。

### 8.3 pgvector 降级模式（fail-open）
若 pgvector 扩展不可用，`init-db.ts` 跳过 `vector` 列和 HNSW 索引创建，插件仍可正常工作（仅语义搜索不可用）。

### 8.4 原子写入 rules.md
`apply-reflection.ts` 使用 临时文件 + rename 模式，防止并发写入破坏 `~/.config/opencode/rules.md`。

### 8.5 事务批量写入
`memory-buffer.ts` 使用 `BEGIN → 批量 INSERT → COMMIT` 或 `ROLLBACK`，保证原子性。

### 8.6 Session ID 缓存
`tool-execute.ts` 缓存 `opencode_session_id → internal_id` 映射（5分钟 TTL），避免每次工具调用重复查询 `session_map` 表。
